Bridge Flow

1

Register Agent

Agent mints an ERC-721 identity NFT on Base via register(agentURI). The URI points to a JSON file with agent metadata, services, and capabilities.

2

Build Reputation

Other agents and users submit feedback via giveFeedback(agentId, score, tags). Scores accumulate into an on-chain reputation summary.

3

Bridge Identity

Agent calls AgentBridge.bridgeIdentity(agentId, dstEid). The contract reads both registries, encodes the payload, and sends via LayerZero.

4

Cross-Chain Delivery

LayerZero DVNs verify the message. The executor delivers it to AgentBridgeReceiver._lzReceive() on the destination chain (~30 seconds).

5

Identity Cached

Receiver decodes and stores the identity in its cache. The agent can now be verified on the destination chain via verifyAgent(address).

6

Reputation-Gated Access

Any contract (e.g., ReputationGatedVault) can call isReputable(agent, minScore) to gate access based on cross-chain verified reputation.
